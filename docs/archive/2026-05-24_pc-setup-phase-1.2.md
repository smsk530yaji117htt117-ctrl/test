## 🔹 ステップ 2：夕方のダイジェストRoutineを作成

朝と同じ仕組みを17:30に動かすものを作ります。

### 2-1. 新規Routine作成画面を開く
- Routines画面の右上の「**+ New Routine**」または「**新規作成**」ボタンをクリック
- 種類を聞かれたら「**Remote**」（クラウド実行）を選ぶ

### 2-2. 基本設定を入力

| 項目 | 入力する内容 |
|---|---|
| 名前（Name） | `PersonalOS 夕方のダイジェスト` |
| トリガー（Trigger） | 毎日 `17:30 JST` |
| コネクター（Connectors） | **Notion** を追加 |

### 2-3. 指示欄にプロンプトを貼り付け
下のコードブロックの中身を**全部コピー**して、指示欄（Instruction / Prompt 欄）に貼ります。

```
Notion の AI Handoff データベース（ID: f91723343d1b4fed91127cda97adbe59）を Notion コネクター経由で読み取り、ダイジェストページにコメントを追加してください。

手順：

1. Status が「Ready」「Error」「Blocked」の行を最大3件取得する
2. 各行の Task名 / To AI / Updated日付 を取得し、継続日数（今日との差）を計算する
3. 以下のページにコメントを追加する：
   ページ ID: 36a5ae2b8d6a81ee8b46e86c7941058f

コメント本文（rich_text）に以下を含める：
- 冒頭に user mention（user_id: 173d872b-594c-81b4-af4a-000262688c71）を入れる
- その後に以下のテキストを続ける：

🌆 夕方のAIダイジェスト [今日の日付 例: 2026-05-24]

判断待ち: [件数]件
1. [Task名]（[継続日数]日継続中）
   次AI: [To AI]
2. ...

4. コメント追加が成功したら「夕方のダイジェスト送信完了」と出力する
5. 失敗したらエラーメッセージを出力する

文言ルール：「未対応」「対応してください」は使わない。「判断待ち」「継続中」を使う。
```

### 2-4. 保存
- 右上の「**保存**」または「**Save**」をクリック
- 一覧画面に戻り、新しく「PersonalOS 夕方のダイジェスト」が増えていることを確認

✅ **増えたら次へ**

## 🔹 ステップ 3：Deep Research 昼Routineを作成

これが今日のメインです。「深掘り：〇〇」と書かれた Notion タスクを Claude が深掘りして結論を返してくれる仕組みです。

### 3-1. 新規Routine作成
- Routines画面 → 右上の「**+ New Routine**」 → 「**Remote**」を選択

### 3-2. 基本設定

| 項目 | 入力する内容 |
|---|---|
| 名前 | `PersonalOS Deep Research 昼` |
| トリガー | 毎日 `12:00 JST` |
| コネクター | **Notion** を追加 |

### 3-3. 指示欄に下のプロンプトを貼り付け

```
あなたは PersonalOS の Deep Research 担当です。

【データソース】
- AI Handoff DB ID: f91723343d1b4fed91127cda97adbe59
- ダイジェストページ ID: 36a5ae2b8d6a81ee8b46e86c7941058f
- 矢嶋勇輝 user_id: 173d872b-594c-81b4-af4a-000262688c71

【手順】
1. AI Handoff DB から以下条件の行を取得:
   - Status = Ready
   - Task名が「深掘り：」または「深掘り:」で始まる
   - 最大3件、古い順に優先

2. 対象が0件なら「Deep Research 対象なし」と出力して終了

3. 各行について以下を実行:
   a. Task名から「深掘り：」プレフィックスを除いたテーマを抽出
   b. テーマを Web 検索（3-7クエリ、信頼できるソースを優先）
   c. 内部的に深く分析（出力には全部出さない）:
      - 事実 / データ
      - 異なる視点や対立論点
      - 実務的な示唆
      - 不確実性や限界
   d. 分析を踏まえて【結論ファースト】の形に組み立てる
   e. 元 Handoff の Notes フィールドに以下フォーマットで追記（上書き禁止）:

      ## Deep Research 結果 [YYYY-MM-DD HH:MM]

      ### 結論
      [80-150字。一目で要旨が掴める形。判断材料として読める]

      ### 根拠（3-5ポイント）
      - [ポイント1：1-2行]
      - [ポイント2：1-2行]
      - [ポイント3：1-2行]

      ### 留意点
      - [不確実性・限界・反対論があれば。なければ「特になし」]

      ### 参考 URL
      - [URL 1]
      - [URL 2]

   f. 元 Handoff の Status を Done に変更
   g. 元 Handoff の Completed 日付を今日に設定

4. 全処理完了後、ダイジェストページ（ID: 36a5ae2b8d6a81ee8b46e86c7941058f）にコメント投稿:
   コメント本文（rich_text）冒頭に user mention（user_id: 173d872b-594c-81b4-af4a-000262688c71）を入れ、その後に:

   🔍 Deep Research 結果 [今日の日付]

   完了: [件数]件

   1. [テーマ名]
      結論: [Notes に書いた「結論」セクションをそのまま再掲、80-150字]

   2. [テーマ名]
      結論: [同上]

   詳細は Notion で

5. 「Deep Research 実行完了」と出力

【絶対禁止】
- Notion スキーマの変更
- AI Handoff DB 以外への書き込み
- 元 Handoff の Task名 / Goal / Current State の上書き（Notes 追記のみ）
- API キーや認証情報の表示
- Notes 全体で800字以上にしない
- コメント通知に根拠・留意点・URL を含めない（結論のみ）

【エラー時】
- 個別の行で失敗したら、その行はスキップして次へ
- 部分失敗でも完了した分は完了として処理
- 全件失敗ならダイジェストに「エラーで未完了」とコメント

【文言ルール】
- 「未対応」「対応してください」「放置されています」を使わない
- 「処理中」「結果は元ページ参照」「エラーで未完了」を使う
```

### 3-4. 保存
- 「保存」をクリック → 一覧に「PersonalOS Deep Research 昼」が増えていることを確認

✅ **増えたら次へ**

## ❓ うまくいかなかった時

### A. Routineが「今すぐ実行」できない / エラーが出る
→ エラーメッセージをスマホで撮ってClaudeチャット（私）に送ってください。原因を切り分けます。

### B. Notion に通知が来ない
→ スマホでNotionアプリを開き、ダイジェストページに新しいコメントがあるか確認。あれば通知設定の問題（Notionアプリのプッシュ通知許可など）。なければRoutine実行が失敗している。

### C. Notes に結果が書かれていない
→ Routineの実行ログを見て、どこで止まっているか確認。Claudeチャット（私）にログのスクショを送ってください。

### D. その他
→ Claudeチャット（私）にそのまま「〇〇でつまってる」と送ってください。
