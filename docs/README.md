# PersonalOS ドキュメント（docs/）

このフォルダは PersonalOS の「人が読むドキュメント」の正本です。
設計メモ・運用ルール・手順書・一回限りの指示書・完了記録は、すべてここ（GitHub の markdown）に置きます。

## なぜ二層なのか
Notion を「機械が読み書きする状態」と「人が読むドキュメント」の両方に使うと、ドキュメントは作るたびに溜まり、手で片付ける前提になります。これは「手間・意志力ゼロ化」という PersonalOS の設計思想に反します。そこで役割で置き場所を固定します。

| 層 | 中身 | 置き場 |
|---|---|---|
| 状態・記録 | 各データベース（Handoff / Consensus / 行動カルテ / 習慣 / Bridge / 体重 等）、セッションログ、引き継ぎ書、ダッシュボード、自己理解の記録層（今後追加） | Notion |
| 説明書・設計 | 運用ルール、制約リスト、Status 意味分け、ロールバック / PAT 手順、Dispatcher / ダイジェスト等の仕様、Phase ドキュメント、一回限りの指示書、監査レポート | repo（このフォルダ） |

データベースは自由ページのように散らからないため Notion に残します。ドキュメントは版管理され Claude Code が直接読める repo に集約します。

## フォルダ構成
- `conventions/` … 生きた運用ルール（制約リスト・Status 意味分け 等）。今も参照するもの
- `specs/` … 設計・仕様（Dispatcher / ダイジェスト / 各 Phase の設計）
- `runbooks/` … 手順書（再利用する作業手順・ロールバック・PAT 更新 等）
- `archive/` … 完了・陳腐化したもの（読むだけ。済んだ手順や指示書）

## 運用ルール
- 作成・編集は Claude Code がブランチ + PR で行う。main への直 push 禁止、自動マージ禁止、マージは矢嶋さんの承認のみ
- 矢嶋さんは GitHub を直接触らない。必要時は窓口OS（Claude チャット）が読み出して要約する
- 新しいドキュメントは Notion ではなくこのフォルダに作る（これが恒久ルール）

## ファイル名規約
- 日付があるもの: `YYYY-MM-DD_短い英語スラグ.md`（例 `2026-05-24_pc-setup-phase-1.2.md`）
- 継続ドキュメント: `短い英語スラグ.md`（例 `cloud-routine-constraints.md`）
