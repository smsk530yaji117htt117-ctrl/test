# テーマ台帳（コンテンツ自動蓄積パイプライン）

記事生成ルーティンはこの表を正本として「レビュー≠却下 かつ 進捗=未着手」の行を1件選び、
草稿を生成したら該当行の進捗を「草稿済み」に更新する（重複防止）。
テーマ補充ルーティン（週1）は末尾に「出所=自動／レビュー=未確認／進捗=未着手」で5行追記する。

列の意味:

- **出所**: 手動 / 自動（補充ルーティンが足したもの）
- **レビュー**: 採用 / 未確認 / 却下。記事生成は **却下を必ずスキップ**し、未確認・採用は対象にする。
- **進捗**: 未着手 / 草稿済み
- **slug**: 草稿ファイル名 `YYYY-MM-DD-slug.md` に使う英小文字ハイフンの識別子（人手可読・台帳内で一意）。
  空欄の行はタイトル由来の安定ハッシュ `theme-xxxxxxxx` に自動フォールバックする。

矢嶋さんの操作は任意で「却下」を付けるだけ（暴走の停止）。手動追加は「採用」にすれば確実に記事化される。
