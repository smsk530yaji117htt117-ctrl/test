# 00_plan.md — 社内アワード設計 調査計画

## 目的
正社員約2,500名・参加約1,000名の社内アワードを、**成果よりプロセスにこだわる行動変容**を真のゴールとして成功させる。残り約1か月（提出期限2026-07-18）。最終成果物は「今年使える最小構成」を確定する根拠付き意思決定材料。

## 与件（=検証対象の仮説。先に答えを再生産しない）
- 暫定3軸配点：プロセス卓越60 / 付加価値25 / 変容性15 → **妥当性は検証対象**
- 一次審査AI化希望。昨年約900件（Googleスライド＋音源）＋採点結果あり → **正解ラベルとして使ってよいかは検証対象**
- 二次=役員面接、優秀者は動画化・表彰式上映
- 個人戦中心、本部編成の異職種5〜6名チーム（任意・業務外）あり
- 個人賞/チーム賞の採点・公平性が未確定

## 論点と完了条件
- [x] t1 組織文化変革 — 完了。単発イベントは弱い。物語+ロールモデル可視化+高頻度ピア承認+受賞事例の型化。サプライズ表彰は逆効果
- [x] t2 動機づけ — 完了。統制的報酬はcrowding-out（HBS皆勤賞で高貢献者−6〜8%）。象徴的承認＋all-can-win＋プロセス称賛。非受賞者900名超への影響が最大リスク
- [x] t3 評価設計・心理測定 ★優先 — 完了。分析的ルーブリック＋BARS＋校正会議。結果バイアス/ハローが最大脅威。プロセス60は事後確定推奨
- [x] t4 AI審査 ★最優先 — 完了。一致≠妥当性。AIは落選単独確定せず証拠抽出・仮採点・振り分けに限定。順序ランダム化/長さ統制/匿名化/選択的予測/PoLL。昨年点の正解ラベル化は危険
- [x] t5 提出物設計 — 完了。提出最大2件・採点は最良1件。8±2枚/~1,500字。デザインは採点外(ハロー)。事実/解釈分離。AI利用は開示必須・捏造失格
- [x] t6 チーム運営 — 完了。社会的手抜き条件が揃う→貢献可視化・個人帰属・小ユニット。心理的安全性が調整変数。役割軽量分散＋非同期最小運用
- [x] t7 公平性（職種横断・反実仮想）★優先 — 完了。手続き的公正が最大レバー／AI単独足切り不可／機械的補正は逆効果
- [x] t8 1,000件規模の運営 — 完了。ファネル1000→250-300→40-50→10-25。全件2名査読(Zスコア標準化)+ボーダー第3査読。審査ノイズ(NeurIPS κ=0.23)前提
- [x] t9 効果測定 — 完了。提出率は遅行指標。RPPF型5時点測定+Kirkpatrick L3。L1満足度偏重を避ける

## 分岐ルール（判明したら計画を書き換える）
- 昨年データに項目別点数が無い→層化抽出で150〜250件の人手再採点・校正データを作る前提に組み直す
- 審査員間一致が低い→昨年点を正解ラベルにしない。昨年資料は事例集化
- 職種別点差大→成果絶対額を弱め、役割・権限・制約への価値増幅を評価
- AI×人間上位一致低い→AIを証拠抽出・優先順位付け・振り分けに限定
- 音源品質低い→音源は補助、スライドを主評価
- 機密を外部AIに入れられない→匿名化・閉域前提を主案
- 検証時間不足→AIは順位/落選を単独確定しない設計
